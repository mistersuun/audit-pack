# rj Sheet Documentation

**Sheet Name:** `rj`
**Dimensions:** 69 rows x 95 columns
**Purpose:** Main R.J. (Rapport Journalier / Daily Report) - Summary of all daily operations

---

## Overview

The `rj` sheet is the **main summary report** that consolidates:
- Revenue from all departments (rooms, food, beverage, etc.)
- Month-to-date comparisons
- Budget variances
- Room statistics
- Deposit summaries
- Credit card totals

This is the primary report printed for management review.

## Key Formulas & Connections

### Data Sources

| rj Cell | Source | Description |
|---------|--------|-------------|
| B51 (Argent recu) | jour!BU[day] | Cash received |
| B52 (Remb. Client) | jour!BV[day] | Client refunds |
| B53 (Remb. Gratuité) | jour!BW[day] | Gratuity refunds |
| B54 (S & D de caisse) | jour!CA[day] | Surplus/Deficit |
| B55 (Depot Du reception) | jour!BY[day] | Reception DueBack |
| B64 (Transfert C.C) | transelect totals | Credit card totals |

### Balance Calculation
```
Balance Fermeture = Balance Ouverture + Total Revenues + Total Credits + Adjustments
```

## Implementation Notes

### For WebApp:
1. **Display Only** - This is primarily a summary/report sheet
2. **Auto-Calculate** - All values come from other sheets
3. **Print Function** - Should generate printable PDF
4. **Month-to-Date** - Calculated from sum of daily values

### Dependencies:
- `jour` sheet provides daily transaction data
- `transelect` provides credit card breakdowns
- `controle` provides date and hotel info
- All department sheets feed into totals
